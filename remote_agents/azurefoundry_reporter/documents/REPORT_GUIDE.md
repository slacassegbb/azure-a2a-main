# Report Creation Guide

This document provides guidance on the report types and templates available from the Reporter Agent.

## Available Report Types

### 1. Executive Summary Report

**Purpose:** High-level overview for leadership and stakeholders

**Best Used For:**
- Board presentations
- C-suite briefings
- Stakeholder updates
- Strategic decision support

**Key Sections:**
- Executive Summary (2-3 paragraphs)
- Key Findings (bullet points)
- Recommendations (numbered with rationale)
- Next Steps (action items)

**Example Requests:**
- "Create an executive summary on our Q4 performance"
- "Generate a leadership report on the market expansion opportunity"
- "Write an executive overview of the acquisition target"

---

### 2. Technical Analysis Report

**Purpose:** Detailed technical evaluation with data and metrics

**Best Used For:**
- System architecture reviews
- Performance assessments
- Security audits
- Technology evaluations

**Key Sections:**
- Overview (context and scope)
- Detailed Analysis (by component/area)
- Metrics & Data (tables with benchmarks)
- Risks & Considerations
- Conclusions

**Example Requests:**
- "Create a technical analysis of our microservices architecture"
- "Generate a performance report for the API gateway"
- "Write a security assessment of the authentication system"

---

### 3. Project Status Report

**Purpose:** Progress updates with milestones and blockers

**Best Used For:**
- Weekly/monthly status updates
- Sprint reviews
- Program management
- Milestone tracking

**Key Sections:**
- Progress Summary
- Accomplishments This Period
- Upcoming Milestones (table)
- Issues & Blockers (table with impact/resolution)
- Key Metrics (budget, timeline, resources)
- Outlook

**Example Requests:**
- "Create a project status report for the cloud migration"
- "Generate a weekly update for the development team"
- "Write a milestone report for Phase 2 completion"

---

### 4. Custom Report

**Purpose:** Flexible format for any topic

**Best Used For:**
- Research summaries
- Competitive analyses
- Market reports
- Any non-standard format

**How to Request:**
1. Specify the topic
2. Define the target audience
3. List desired sections
4. Mention any specific data points needed

**Example Requests:**
- "Create a report on AI trends in healthcare with sections on diagnosis, treatment, and operations"
- "Generate a competitive analysis of the top 3 competitors in our space"
- "Write a market research report with executive summary, market size, trends, and recommendations"

---

## Report Quality Guidelines

### What Makes a Good Report

1. **Clear Structure** - Logical flow from overview to details to recommendations
2. **Actionable Insights** - Each section should provide value
3. **Data-Driven** - Include metrics, numbers, and benchmarks where applicable
4. **Audience-Appropriate** - Technical depth matched to the reader
5. **Visual Hierarchy** - Use headers, bullets, tables, and formatting

### Requesting Modifications

After receiving a report, you can ask to:
- Expand specific sections
- Add more data/metrics
- Change the tone (more technical, more executive)
- Add or remove sections
- Include additional sources

---

## Tips for Best Results

1. **Be Specific** - "Create a technical analysis of our AWS costs" is better than "Make a report"
2. **Specify Audience** - "For the CTO" vs "For the engineering team"
3. **Mention Scope** - "Q4 2024" or "Last 6 months"
4. **Request Format** - "Include a table comparing the three options"
